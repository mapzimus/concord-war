# concord-war — interactive web map

A scrollytelling + free-explore web map of the alternate-history
**"East Concord vs. West Concord" civil-war** concept. Companion to the
[Concord civil-war poster](https://github.com/mapzimus/concord-civil-war-map)
(static ARCH-D print).

**Live:** <https://mapzimus.github.io/concord-war/>

## Stack

SvelteKit (`adapter-static`) + MapLibre GL JS + deck.gl + scrollama, deployed to
GitHub Pages.

| Concern | Choice |
|---|---|
| Framework | SvelteKit 2 (Svelte 5 runes), adapter-static |
| Map base | MapLibre GL JS 4 (sepia raster style) |
| Overlays | deck.gl 9 via `@deck.gl/mapbox` MapboxOverlay |
| Scrollytelling | scrollama 3 |
| Tests | Vitest 2 (JS state) + pytest (Python data pipeline) |
| Deploy | GitHub Pages via Action on push to `main` |

## Data

GeoJSON layers under `static/data/` are regenerated by Python from the GeoPackage
in the [companion poster repo](https://github.com/mapzimus/concord-civil-war-map):

```powershell
& 'C:\Users\Calen\miniforge3\condabin\conda.bat' run --no-capture-output -n gis python scripts\build_geojson.py
```

Sources: City of Concord GIS (`gis.concordnh.gov`), USGS NHD, USGS 3DEP,
US Census TIGER 2020. Each layer's `source_url` is in `static/data/manifest.json`.

## Pass A scope

This is **Pass A — Foundation**. It ships:

- 1 story chapter (The Two Banks)
- 1 explorer mode (free zoom/pan, all layers)
- 6 layers (city, river, front line, East/West territories, crossings)
- Sepia 1860s palette
- GitHub Pages deploy on push

Passes B (full 10-chapter story), C (resource drawer + forces dashboard),
and D (3D terrain + animated arcs + portfolio mount) are planned separately.

See `docs/superpowers/specs/2026-06-13-concord-war-web-map-design.md` in the
companion repo for the full design.
